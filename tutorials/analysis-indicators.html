Analysis indicators分析指標

Overview​概要​

This guide provides an overview of the custom indicator examples. These examples serve as a starting point for creating your own indicators. You can use them directly in your projects.このガイドではカスタムインジケーターの例についての概要を提供します。 これらの例は、自分自身のインジケーターを作成するための出発点として役立ちます。 プロジェクトに直接使用することができます。

Available indicators​利用可能なインジケーター​

Below is a list of indicators where each link points to their source code on GitHub.以下は、各リンクがGitHub上のソースコードにポイントするインジケーターのリストです。

  • Average Price平均価格
  • Correlation相関
  • Median Price中央値価格
  • Momentumモメンタム
  • Simple Moving Average単純移動平均
  • Percent Changeパーセント変化
  • Product積
  • Ratio比率
  • Spreadスプレッド
  • Sum合計
  • Weighted Close加重終値

Live demos​ライブデモ​

You can see all the indicators in action on the live demos page. Each indicator has two demos:すべてのインジケーターの動作をライブデモページで見ることができます。 各インジケーターには2つのデモがあります:

  • Helper: shows the recommended method with automatic updates.ヘルパー: 自動更新を伴う推奨方法を示します。
  • Direct calculation: shows the method with a pure function.直接計算: 純粋な関数を用いた方法を示します。

How to use the examples​例の使い方​

The examples are self-contained and not available on a package manager like NPM. Therefore, you have two options for integrating them into your project.例は自己完結型であり、NPMのようなパッケージマネージャーでは利用できません。 したがって、プロジェクトに統合するための2つのオプションがあります。

Option 1: copy the source code​オプション1: ソースコードをコピー​

The simplest way to use an indicator is to copy its source code directly into your project. For example, if you want to use the Moving Average indicator, copy the following files into your project's source tree.インジケーターを使用する最も簡単な方法は、そのソースコードを直接プロジェクトにコピーすることです。 たとえば、移動平均インジケーターを使用したい場合は、以下のファイルをプロジェクトのソースツリーにコピーしてください。

  • indicator-examples/src/indicators/moving-average/moving-average.ts
  • indicator-examples/src/indicators/moving-average/moving-average-calculation.ts
  • indicator-examples/src/helpers/timestamp-data.ts (dependency for the calculation)indicator-examples/src/helpers/timestamp-data.ts（計算の依存関係）

You can then import the applyMovingAverageIndicator helper or the calculateMovingAverageIndicatorValues function directly into your code.次に、applyMovingAverageIndicatorヘルパーまたはcalculateMovingAverageIndicatorValues関数を直接コードにインポートできます。

Option 2: compile the examples​オプション2：例をコンパイルする​

If you prefer to use a compiled JavaScript module, you can build the examples yourself.コンパイルされたJavaScriptモジュールを使用したい場合は、自分で例をビルドできます。

  1. Clone the lightweight-charts repository.lightweight-chartsリポジトリをクローンします。
  2. Build the main library first: 
    npm install
    npm run build:prod
    まずメインライブラリをビルドします： 
    npm install
    npm run build:prod
  3. Navigate to the examples directory, install dependencies, and run the compile script: 
    cd indicator-examples
    npm install
    npm run compile
    例のディレクトリに移動し、依存関係をインストールし、コンパイルスクリプトを実行します： 
    cd indicator-examples
    npm install
    npm run compile
  4. The compiled output will be available in the indicator-examples/compiled folder. You can then copy this folder into your project and import the modules.コンパイルされた出力はindicator-examples/compiledフォルダーに利用可能になります。このフォルダーをプロジェクトにコピーし、モジュールをインポートできます。

How to add indicator​インジケーターの追加方法​

There are two distinct approaches to applying these indicators to your chart.これらのインジケーターをチャートに適用するための2つの異なるアプローチがあります。

  • Using a helper function that creates the indicator series and automatically keeps it in sync with the source series' data.ヘルパー関数を使用してインジケーターシリーズを作成し、ソースシリーズのデータと自動的に同期させる方法。
  • Using a pure function to directly calculate the indicator data from a static dataset.純粋な関数を使用して、静的データセットから直接インジケーターデータを計算する方法。
tip

We recommend using the Helper function for its simplicity and automatic data synchronization.シンプルさと自動データ同期のために、ヘルパー関数の使用をお勧めします。

Helper function (recommended)​ヘルパー関数（推奨）​

Each indicator includes an apply… function (e.g., applyMovingAverageIndicator). This is the preferred and easier method.各インジケーターには、apply…関数（例：applyMovingAverageIndicator）が含まれています。これは推奨される簡単な方法です。

This function takes the source series API object itself (not the data) and the options. It handles everything for you:この関数は、ソースシリーズのAPIオブジェクト自体（データではなく）とオプションを受け取ります。 すべてを自動で処理します：

  • Creates the new indicator series.新しいインジケーターシリーズを作成します。
  • Performs the initial calculation.初期計算を実行します。
  • Automatically listens for data changes on the source series and recalculates the indicator whenever the source data is updated.ソースシリーズのデータ変更を自動的にリスンし、ソースデータが更新されるたびにインジケーターを再計算します。

Example​例​

The example below shows how to add an Exponential Moving Average (EMA) with the helper function.以下の例では、ヘルパー関数を使用して指数移動平均（EMA）を追加する方法を示します。

import { createChart, CandlestickSeries, LineStyle } from 'lightweight-charts';
import { applyMovingAverageIndicator } from './indicators/moving-average/moving-average';
import { symbolData } from './my-data-source';

const chart = createChart(document.body);
const mainSeries = chart.addSeries(CandlestickSeries);
mainSeries.setData(symbolData.slice(0, 100)); // Set initial data

// 1. Apply the indicator directly to the source series
const emaSeries = applyMovingAverageIndicator(mainSeries, {
    length: 10,
    source: 'close',
    smoothingLine: 'EMA',
});

// 2. (Optional) Customize the new indicator series
emaSeries.applyOptions({
    color: 'orange',
    lineWidth: 2,
    lineStyle: LineStyle.Dotted,
});

// Now, when we update the mainSeries, the emaSeries will update automatically
setInterval(() => {
    const nextBar = getNextRealTimeBar();
    mainSeries.update(nextBar); // The EMA series will update itself
}, 1000);

The apply… helper attaches a lightweight ISeriesPrimitive to the source series. This primitive subscribes to the series' data changes. When a change is detected, it refetches the data, runs the calculation, and updates the indicator series automatically.このapply…ヘルパーは、ソースシリーズに軽量のISeriesPrimitiveを接続します。 このプリミティブは、シリーズのデータ変更にサブスクライブします。 変更が検出されると、データを再取得し、計算を実行し、インジケーターシリーズを自動的に更新します。

This approach is more robust, requires less code, and is the recommended way to use these examples.このアプローチはより堅牢で、コードが少なくて済み、これらの例を使用するための推奨方法です。

Direct calculation​直接計算​

Each indicator includes a calculate… function (e.g., calculateMovingAverageIndicatorValues). This is a pure function that takes your series data and a set of options as input and returns an array of calculated data points for the indicator.各インジケーターにはcalculate…関数（例：calculateMovingAverageIndicatorValues）が含まれています。 これは純粋な関数で、あなたのシリーズデータとオプションのセットを入力として受け取り、インジケーターの計算されたデータポイントの配列を返します。

This method is useful if you have a static dataset or want full control over when the indicator is recalculated.このメソッドは、静的データセットを持っている場合や、インジケーターの再計算のタイミングを完全に制御したい場合に便利です。

Example​例​

The example below shows how to add a Simple Moving Average (SMA).以下の例では、単純移動平均（SMA）を追加する方法を示します。

import { createChart, LineSeries, CandlestickSeries } from 'lightweight-charts';
import { calculateMovingAverageIndicatorValues } from './indicators/moving-average/moving-average-calculation';
import { symbolData } from './my-data-source';

const chart = createChart(document.body);
const mainSeries = chart.addSeries(CandlestickSeries);
mainSeries.setData(symbolData);

// 1. Calculate the indicator data from the source data
const smaData = calculateMovingAverageIndicatorValues(symbolData, {
    length: 20,
    source: 'close',
});

// 2. Create a new series for the indicator
const smaSeries = chart.addSeries(LineSeries, {
    color: 'blue',
    lineWidth: 2,
});

// 3. Set the calculated data on the new series
smaSeries.setData(smaData);
warning

This approach is not reactive. If you update the mainSeries with new data (e.g., from a real-time feed), the smaSeries will not update automatically. You are responsible for manually recalculating the indicator and calling smaSeries.setData() again.このアプローチは反応的ではありません。mainSeriesを新しいデータ（例：リアルタイムフィードから）で更新すると、smaSeriesは自動的に更新されません。 インジケーターを手動で再計算し、再度smaSeries.setData()を呼び出す責任があります。
