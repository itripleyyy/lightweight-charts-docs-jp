Integrating ARIA roles and descriptive content for enhanced accessibilityアクセシビリティ向上のためのARIAロールと説明コンテンツの統合

Accessibility in web development extends beyond just accommodating keyboard-only users. Many users with varying abilities make use of assistive technologies like screen readers to effectively interact with web applications. ARIA (Accessible Rich Internet Applications) roles and states offer a powerful toolkit to improve this interaction by conveying information about the behavior and purpose of interface components.ウェブ開発におけるアクセシビリティは、キーボードのみのユーザーへの配慮を超えています。さまざまな能力を持つ多くのユーザーが、スクリーンリーダーのような支援技術を利用して、ウェブアプリケーションと効果的に対話しています。 ARIA （アクセシブルリッチインターネットアプリケーション）ロールと状態は、インターフェースコンポーネントの動作や目的に関する情報を伝えることで、この対話を改善するための強力なツールキットを提供します。

In the context of our line chart, we'll be looking at how to implement ARIA attributes to further enhance its accessibility in addition to generating descriptive content for our graph.私たちの折れ線グラフの文脈では、グラフの説明コンテンツを生成することに加えて、アクセシビリティをさらに向上させるためにARIA属性を実装する方法を見ていきます。

info

In this tutorial we will only make use of the aria-live, aria-label, and aria-hidden attributes. You can explore a [more comprehensive list of attributes on this MDN page] (https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes).このチュートリアルでは、aria-live、aria-label、およびaria-hidden属性のみを使用します。より包括的な属性のリストは、このMDNページで確認できます。 (https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes)。

Use of ARIA attributes in the chart​チャートにおけるARIA属性の使用​

aria-live​

To provide real-time updates about the chart to assistive technologies, we use the aria-live attribute. It accepts a few potential values, although "polite" and "assertive" are the most common.チャートに関するリアルタイムの更新を支援技術に提供するために、aria-live属性を使用します。いくつかの潜在的な値を受け入れますが、"polite"と"assertive"が最も一般的です。

With aria-live="polite", updates are presented at the user's next convenient opportunity, such as when they stop typing or when a task is completed. With aria-live="assertive", updates are presented immediately.aria-live="polite"を使用すると、更新はユーザーの次の便利な機会に提示されます。たとえば、ユーザーが入力を停止したときやタスクが完了したときです。aria-live="assertive"を使用すると、更新は即座に提示されます。

Our chart uses the "assertive" value because the description should be read out based on a user action (keyboard action):私たちのチャートは、ユーザーのアクション（キーボードアクション）に基づいて説明が読み上げられるべきなので、"assertive"の値を使用しています。

<div id="chart" aria-live="assertive" tabindex="0"></div>

aria-label​

The aria-label attribute is used to specify a string that labels the current element. It's useful when there isn't any text content that describes this element.aria-label属性は、現在の要素にラベルを付ける文字列を指定するために使用されます。この要素を説明するテキストコンテンツがない場合に便利です。

On our chart, it could read something like this:私たちのチャートでは、次のように読み上げられる可能性があります：

<div
    id="chart"
    aria-live="polite"
    aria-label="interactive line chart"
    tabindex="0"
></div>

aria-hidden​

The aria-hidden attribute is used to hide irrelevant or redundant information from assistive technologies. Its value is either "true" or "false".aria-hidden属性は、関連性のないまたは冗長な情報を支援技術から隠すために使用されます。その値は「true」または「false」です。

For example, if our chart contained a decorative element with no semantic meaning, we could use aria-hidden="true" to hide it from screen readers:例えば、私たちのチャートに意味のない装飾的な要素が含まれている場合、aria-hidden="true"を使用してスクリーンリーダーから隠すことができます：

<div id="decorative-element" aria-hidden="true"></div>

Adding the ARIA attributes to an existing chart via JavaScript​JavaScriptを介して既存のチャートにARIA属性を追加する​

<!-- Template for HTML elements added to the chart container for the A11y improvements -->
<template id="a11y-helpers">
    <div tabindex="-1" role="alert" aria-live="assertive"></div>
</template>

<script>
    function addAriaAttributesAndAlerter(chart) {
        const containerEl = chart.chartElement().parentElement;
        if (!containerEl) return;
        // make focusable
        containerEl.tabIndex = 0;
        containerEl.style.position = 'relative';
        containerEl.ariaLabel =
            'Line plot of Accessibility stock price. Press the H key to display the available interaction keys.';
        chart.chartElement().ariaHidden = 'true';

        const templateElement = document.getElementById('a11y-helpers');
        const clone = templateElement.content.cloneNode(true);
        containerEl.appendChild(clone);
    }
</script>

Generating a description of the chart​チャートの説明を生成する​

In addition to ARIA roles, providing a textual description for our charts helps all users, especially those relying on screen readers or other assistive technologies.ARIAロールに加えて、私たちのチャートにテキストの説明を提供することは、すべてのユーザー、特にスクリーンリーダーやその他の支援技術に依存しているユーザーに役立ちます。

You may generate a description of a chart by applying domain-specific knowledge. It's beneficial to outline the general trend or patterns of the data, highlight any notable points or anomalies, and summarize the implications of the data.ドメイン固有の知識を適用することで、チャートの説明を生成できます。 データの一般的な傾向やパターンを概説し、目立つポイントや異常を強調し、 データの含意を要約することが有益です。

You can add a descriptive section to the chart:チャートに説明セクションを追加できます：

<div id="chart-description" aria-live="assertive">
    <!-- The content here should be dynamically generated based on the chart data -->
</div>

You can use JavaScript to change the content inside this div whenever the chart data is updated:チャートのデータが更新されるたびに、このdiv内のコンテンツを変更するためにJavaScriptを使用できます：

const descriptionElement = document.getElementById('chart-description');
descriptionElement.textContent = generateDescription(mainSeries.data());

Here, generateDescription(data) would be a function that you write to translate the chart's data into human-readable insights. The function would vary greatly based on what the charts represent and how much detail you wish to provide.ここで、generateDescription(data)は、チャートのデータを人間が読みやすい洞察に変換するためにあなたが書く関数です。関数は、チャートが何を表しているか、提供したい詳細の量によって大きく異なります。

The example describes the chart based on its first and last visible data points in addition to the highest and lowest points displayed. This is used to generate a description like this:この例では、最初と最後の可視データポイントに加えて、表示される最高点と最低点に基づいてチャートを説明します。これを使用して、次のような説明を生成します：

The first price is $679.10 at Wed Sep 19 2018. The last price is $555.37 at Wed May 15 2019. The actual change in price was -$123.73, corresponding to a percentage change of -18.22%. The lowest price was $32.76 at Fri Dec 21 2018. The highest price was $951.33 at Sun Mar 24 2019.最初の価格は2018年9月19日水曜日の$679.10です。最後の価格は2019年5月15日水曜日の$555.37です。実際の価格変動は-$123.73で、パーセンテージ変動は-18.22%に相当します。最低価格は2018年12月21日金曜日の$32.76でした。 最高価格は2019年3月24日日曜日の$951.33でした。

The following code could be used as a starting point for generating chart descriptions:チャートの説明を生成するための出発点として、次のコードを使用できます：

function formatDate(time) {
    return new Date(time * 1000).toDateString();
}

function formatValue(value) {
    return `${value < 0 ? '-' : ''}$${Math.abs(value).toFixed(2)}`;
}

function getStats(data) {
    const stats = {
        start: data[0],
        close: data[data.length - 1],
        low: data[0],
        high: data[0],
    };

    for (const point of data) {
        if (point.value > stats.high.value) {
            stats.high = point;
        }
        if (point.value < stats.low.value) {
            stats.low = point;
        }
    }

    return stats;
}

function getVisibleSeriesData(chart, series) {
    const timeScale = chart.timeScale();
    const visibleRange = timeScale.getVisibleLogicalRange();
    const data = [];
    for (let i = Math.round(visibleRange.from); i <= visibleRange.to; i++) {
        const d = series.dataByIndex(i, 0);
        if (d !== null) {
            data.push(d);
        }
    }
    return data;
}

function describeFinanceChart(data) {
    if (!data || data.length === 0) {
        return 'The data set is empty.';
    }

    const stats = getStats(data);

    const firstPrice = `The first price is ${formatValue(
        stats.start.value
    )} at ${formatDate(stats.start.time)}.`;
    const lastPrice = `The last price is ${formatValue(
        stats.close.value
    )} at ${formatDate(stats.close.time)}.`;

    const actualChange = stats.close.value - stats.start.value;
    const percentChange = (actualChange / stats.start.value) * 100;

    const changeDescription = `The actual change in price was ${formatValue(
        actualChange
    )}, corresponding to a percentage change of ${percentChange.toFixed(2)}%.`;

    let lowHigh = '';
    if (
        stats.low.time !== stats.start.time &&
        stats.low.time !== stats.close.time
    ) {
        lowHigh += `The lowest price was ${formatValue(
            stats.low.value
        )} at ${formatDate(stats.low.time)}.`;
    }
    if (
        stats.high.time !== stats.start.time &&
        stats.high.time !== stats.close.time
    ) {
        lowHigh += ` The highest price was ${formatValue(
            stats.high.value
        )} at ${formatDate(stats.high.time)}.`;
    }

    return `${firstPrice} ${lastPrice} ${changeDescription} ${lowHigh}`.trim();
}

Semantic HTML​セマンティックHTML​

Using Semantic HTML elements offers several benefits. Firstly, they enhance the accessibility of web content as they provide specific meaning to the browser and assistive technology like screen readers, helping them understand the content's structure and purpose. This is crucial for users with disabilities.セマンティックHTMLの要素を使用することにはいくつかの利点があります。まず、これによりウェブコンテンツのアクセシビリティが向上します。ブラウザやスクリーンリーダーなどの支援技術に特定の意味を提供し、コンテンツの構造や目的を理解するのに役立ちます。これは、障害のあるユーザーにとって重要です。

It is suggested that the container provided to the createChart method should use a semantic element such as <figure> instead of a generic element like div.createChartメソッドに提供されるコンテナは、<figure>のようなセマンティック要素を使用し、divのような一般的な要素を使用しないことが推奨されます。

<figure id="chart"></figure>
