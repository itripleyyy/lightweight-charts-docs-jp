5.0.9​

Enhancements強化

  • Added rightOffsetPixels option to HorzScaleOptions, allowing margin space from the right side of the chart to be set in pixels. This option takes precedence over rightOffset and ensures consistent pixel offset when using fitContent on charts with different amounts of data. The pixel-based offset remains consistent during zoom operations. (PR #1957)HorzScaleOptionsにrightOffsetPixelsオプションを追加し、チャートの右側からのマージンスペースをピクセル単位で設定できるようにしました。このオプションはrightOffsetよりも優先され、異なるデータ量のチャートでfitContentを使用する際に一貫したピクセルオフセットを保証します。ピクセルベースのオフセットはズーム操作中も一貫性を保ちます。(PR #1957)
  • Added pop method to series API that removes a specified number of data points from the end of the series and returns the removed data. (PR #1949, fixes #1518, contributed by @tpunt)popメソッドをシリーズAPIに追加し、シリーズの末尾から指定された数のデータポイントを削除し、削除されたデータを返します。(PR #1949、修正 #1518、寄稿者 @tpunt)
  • Enhanced takeScreenshot method with two new optional parameters:
    • addTopLayer - includes top layer primitives in the screenshot (default: false)addTopLayer - スクリーンショットにトップレイヤーのプリミティブを含めます（デフォルト：false）

When enabled (default: true), the chart will adjust its scale to ensure markers are fully visible. (PR #1940, contributed by @zbinlin)SeriesMarkersOptionsにautoScaleオプションを追加し、マーカーが価格スケールの自動スケーリング計算に含まれるかどうかを制御できるようにしました。有効にすると（デフォルト：true）、チャートはマーカーが完全に表示されるようにスケールを調整します。(PR #1940、寄稿者 @zbinlin)

5.0.0​

Version 5.0.0 represents a significant milestone in the evolution of Lightweight Charts™, delivering on our commitment to keep the library truly "lightweight". Despite adding numerous new features, improvements, and fixes, we've managed to reduce the bundle size by up to 16%, bringing the base bundle size down to just 35kB. This remarkable reduction was achieved through enhanced tree-shaking capabilities, modernized architecture, and careful optimization of core features.バージョン5.0.0は、Lightweight Charts™の進化における重要なマイルストーンを表しており、ライブラリを真に「軽量」に保つという私たちのコミットメントを実現しています。多くの新機能、改善、修正を追加しながらも、バンドルサイズを最大16%削減し、基本バンドルサイズをわずか35kBにまで引き下げることに成功しました。この驚くべき削減は、強化されたツリーシェイキング機能、近代化されたアーキテクチャ、コア機能の慎重な最適化を通じて達成されました。

This release introduces highly requested features like multi-pane support and new chart types. It also modernizes the codebase and improves its architecture to set a foundation for future enhancements without compromising on size.このリリースでは、マルチペインサポートや新しいチャートタイプなど、非常にリクエストの多かった機能が導入されています。また、コードベースを近代化し、将来の拡張のための基盤を整えるためにアーキテクチャを改善していますが、サイズには妥協していません。

Major Updates主な更新

Pluginsプラグイン

Developers can now leverage the power of Plugins in Lightweight Charts. Two types of Plugins are supported -  Custom Series and Drawing Primitives, offering the ability to define new series types and create custom visualizations, drawing tools, and annotations.開発者は、Lightweight Chartsでプラグインの力を活用できるようになりました。サポートされているプラグインの種類は2つあり、 カスタムシリーズ と 描画プリミティブがあり、新しいシリーズタイプを定義し、カスタムビジュアライゼーション、描画ツール、および注釈を作成する能力を提供します。

With the flexibility provided by these plugins, developers can create highly customizable charting applications for their users.これらのプラグインによって提供される柔軟性を活用することで、開発者はユーザーのために非常にカスタマイズ可能なチャートアプリケーションを作成できます。

To get started with plugins, please refer to our Plugins Documentation for a better understanding of what is possible and how plugins work. You can also explore our collection of plugin examples (with a preview hosted here) for inspiration and guidance on implementing specific functionality.プラグインの使い始めるためには、プラグインドキュメントを参照して、可能なことやプラグインの動作についてより良く理解してください。また、特定の機能を実装するためのインスピレーションとガイダンスを得るために、私たちのプラグイン例（ここにホストされたプレビュー付き）を探索することもできます。

The horizontal scale is no longer restricted to only time-based values.
